Todos los cambios importantes de PG Lista 3 AI deberán registrarse en este documento.

# Versionado

# Cómo actualizar este archivo

Cada cambio importante deberá registrarse inmediatamente después de ser implementado.

Nunca esperar al final del desarrollo para actualizar el changelog.

Cada entrada deberá indicar:

* Fecha.
* Versión.
* Tipo de cambio.
* Descripción clara.

# Reglas

No eliminar versiones anteriores.

No modificar el historial publicado.

Registrar únicamente cambios relevantes.

Mantener el orden cronológico.

---

# Objetivo

El CHANGELOG será el historial oficial del proyecto.

Permitirá conocer la evolución del sistema, facilitar auditorías, simplificar el mantenimiento y ayudar a cualquier desarrollador o IA a comprender qué ha cambiado entre versiones.
