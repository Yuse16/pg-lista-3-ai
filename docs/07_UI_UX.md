# User Interface & User Experience

Proyecto: PG Lista 3 AI

Versión: 1.0

---

# Objetivo

Diseñar una aplicación extremadamente rápida, limpia, profesional e intuitiva.

La interfaz deberá ayudar al vendedor a trabajar mejor.

Nunca deberá distraerlo.

Nunca deberá sentirse pesada.

---

# Filosofía

La interfaz debe transmitir:

Profesionalismo.

Orden.

Rapidez.

Claridad.

Confianza.

Tecnología.

Cada pantalla deberá ayudar al usuario a tomar decisiones.

No únicamente mostrar información.

---

# Inspiración

La interfaz deberá inspirarse en productos modernos como:

* Linear
* Notion
* Vercel Dashboard
* Stripe Dashboard
* Raycast
* Arc Browser
* GitHub
* Supabase Studio

No copiar.

Inspirarse.

---

# Estilo visual

Minimalista.

Moderno.

Elegante.

Espacios amplios.

Mucho aire entre componentes.

Sin saturación.

Sin colores innecesarios.

# Dashboard

Debe ser la pantalla más importante.

Al abrir la aplicación el usuario deberá entender su día en menos de cinco segundos.

Debe incluir:

Resumen del día.

Seguimientos.

Clientes prioritarios.

Cotizaciones.

Ventas.

Indicadores.

Alertas.

Actividad reciente.

Recomendaciones IA.

---

# Sidebar

Siempre visible.

Muy limpia.

Secciones:

Dashboard

Clientes

Prospectos

Seguimientos

Agenda

Ventas

Reportes

Configuración

---

# Cards

Cada card deberá mostrar únicamente información importante.

Ejemplo:

Clientes activos

248

+18 este mes

No llenar cards con texto innecesario.

---

# Tablas

Las tablas deberán permitir:

Buscar.

Ordenar.

Filtrar.

Seleccionar varias filas.

Exportar.

Nunca utilizar tablas difíciles de leer.

---

# Formularios

Máximo una columna cuando sea posible.

Agrupar información relacionada.

Validar mientras el usuario escribe.

Nunca obligar a llenar información innecesaria.

---

# IA

La IA siempre deberá estar visible.

No escondida.

Debe existir un botón flotante.

Debe sentirse como un copiloto.

Nunca como una función secundaria.

---

# Responsive

La prioridad será escritorio.

Posteriormente:

Tablet.

Finalmente:

Móvil.

---

# Animaciones

Muy sutiles.

Nunca exageradas.

Utilizar:

Fade.

Slide.

Scale.

Hover.

Transiciones rápidas.

---

# Iconografía

Lucide Icons.

Consistencia en todo el sistema.

No mezclar estilos.

---

---

# Experiencia del usuario

Registrar un prospecto:

Menos de 30 segundos.

Registrar seguimiento:

Menos de 15 segundos.

Buscar cliente:

Menos de 3 segundos.

Abrir ficha:

Instantáneo.

---

# Rendimiento

Toda interacción deberá sentirse inmediata.

Tiempo objetivo:

Menos de 200 ms para acciones locales.

Menos de 1 segundo para consultas remotas.

---

# Accesibilidad

Contraste adecuado.

Navegación con teclado.

Tooltips.

Textos claros.

No depender únicamente del color.

---

# Diseño del Dashboard

El Dashboard deberá responder inmediatamente:

¿Qué debo hacer hoy?

¿Quién necesita seguimiento?

¿Cuánto llevo vendido?

¿Qué oportunidades tengo?

¿Qué clientes están en riesgo?

¿Qué clientes debo contactar primero?

---

# Modo oscuro

El sistema deberá soportar Dark Mode desde la primera versión.

No deberá implementarse posteriormente.

---

# Escalabilidad

Todos los componentes deberán ser reutilizables.

Nunca crear componentes duplicados.

Seguir un Design System.

---

# Regla Suprema

Cada pantalla deberá reducir trabajo al vendedor.

Si una pantalla únicamente muestra datos sin ayudar a tomar decisiones, deberá rediseñarse.

---

# Visión final

Queremos que un asesor comercial pueda abrir PG Lista 3 AI y sentir que está utilizando una herramienta premium, moderna y diseñada específicamente para ayudarlo a vender más.

La interfaz deberá transmitir la misma calidad que productos como Linear, Vercel o Notion, pero adaptada al mundo comercial de Plomería García.
