# FUTURE.md

# Visión Futura del Proyecto

Proyecto: PG Lista 3 AI
Versión: 1.0
Estado: Documento de referencia a largo plazo

---

## Objetivo

Este documento conserva las posibles direcciones futuras de PG Lista 3 AI.

Su propósito es evitar que las oportunidades de crecimiento se pierdan, sin ampliar innecesariamente el alcance del MVP.

Lo registrado aquí:

* No representa trabajo aprobado.
* No constituye una fecha de entrega.
* No debe desarrollarse automáticamente.
* Deberá validarse antes de incorporarse al roadmap.
* No sustituye a `docs/09_ROADMAP.md`.
* No sustituye a `docs/10_TASKS.md`.

---

## Visión futura

PG Lista 3 AI podrá evolucionar desde una herramienta interna para la sucursal Nogalera hasta convertirse en una plataforma comercial inteligente para múltiples sucursales.

La aplicación podría transformarse en una capa de inteligencia sobre Intelisis que permita convertir datos de clientes, cotizaciones, productos y ventas en acciones comerciales concretas.

La visión final es que cada asesor pueda abrir la plataforma y saber:

* Qué clientes debe contactar.
* Qué oportunidades tienen mayor valor.
* Qué cotizaciones requieren seguimiento.
* Qué clientes están dejando de comprar.
* Qué productos puede ofrecer.
* Qué acciones pueden acercarlo a su meta.

---

# Etapa futura 1 — Validación del MVP

Antes de ampliar el proyecto se deberá comprobar:

* Que los vendedores realmente utilicen la aplicación.
* Que registrar clientes y seguimientos sea rápido.
* Que las recomendaciones sean útiles.
* Que disminuyan los clientes olvidados.
* Que aumenten los seguimientos realizados.
* Que exista una mejora medible en ventas Lista 3.
* Que los usuarios comprendan el sistema sin capacitación extensa.

Si estos puntos no se validan, deberán corregirse antes de agregar funciones avanzadas.

---

# Etapa futura 2 — Importación de información

Una vez validado el flujo inicial, el sistema podrá incorporar datos externos.

Posibles fuentes:

* Excel.
* CSV.
* Reportes exportados desde Intelisis.
* Catálogo de productos.
* Historial de clientes.
* Ventas anteriores.
* Cotizaciones.
* Presupuestos y objetivos comerciales.

La importación deberá:

* Detectar duplicados.
* Validar formatos.
* Mostrar errores.
* Permitir revisar la información antes de guardarla.
* Registrar quién realizó la importación.
* Mantener trazabilidad de la fuente.

---

# Etapa futura 3 — Integración con Intelisis

La conexión con Intelisis se evaluará solamente después de validar el MVP y obtener autorización técnica.

Información que podría sincronizarse:

* Clientes.
* Códigos de cliente.
* Clasificación comercial.
* Vendedores.
* Sucursales.
* Productos.
* Familias y categorías.
* Ventas.
* Cotizaciones.
* Existencias.
* Precios.
* Descuentos autorizados.

PG Lista 3 AI no deberá reemplazar Intelisis.

Intelisis seguirá siendo el sistema oficial de operación y PG Lista 3 AI funcionará como herramienta de análisis, organización y seguimiento comercial.

---

# Etapa futura 4 — Agente comercial avanzado

El agente podrá evolucionar para:

* Crear un plan diario personalizado.
* Priorizar clientes automáticamente.
* Detectar clientes en riesgo.
* Identificar cotizaciones olvidadas.
* Sugerir productos complementarios.
* Analizar frecuencia y ticket promedio.
* Detectar patrones de compra.
* Redactar mensajes personalizados.
* Preparar argumentos comerciales.
* Resumir el historial de un cliente.
* Recomendar el mejor siguiente paso.

Toda recomendación deberá:

* Utilizar datos reales.
* Explicar el motivo.
* Mostrar nivel de confianza.
* Permitir revisión humana.
* Evitar acciones automáticas sin autorización.

El sistema podría conectarse con:

* WhatsApp Business.
* Gmail.
* Outlook.
* Telefonía.
* Google Calendar.

Posibles funciones:

# Etapa futura 6 — Inteligencia de productos

El agente podrá conocer mejor el catálogo para:

* Recomendar productos complementarios.
* Buscar alternativas similares.
* Mostrar opciones por formato, estilo o precio.
* Detectar productos con alta existencia.
* Identificar productos de baja rotación.
* Relacionar proyectos con categorías.
* Recomendar combinaciones para baños, cocinas y recubrimientos.

Las recomendaciones deberán respetar:

* Existencia real.
* Compatibilidad.
* Precio vigente.
* Promociones autorizadas.
* Catálogo oficial.

Con suficiente historial, el sistema podría calcular:

* Probabilidad de recompra.

# Criterios para promover una función al roadmap

Una idea podrá pasar de `future.md` a `docs/09_ROADMAP.md` cuando:

1. Resuelva un problema real observado.
2. Exista evidencia de que los usuarios la necesitan.
3. Tenga un beneficio comercial identificable.
4. No distraiga del funcionamiento principal.
5. Sea técnicamente viable.
6. Exista autorización para utilizar los datos necesarios.
7. Tenga criterios de aceptación claros.
8. Se defina una prioridad.

---

# Funciones que no deben adelantarse

No desarrollar prematuramente:

* Integración directa con Intelisis.
* WhatsApp automático.
* Predicciones complejas.
* Aplicación móvil completa.
* Arquitectura multiempresa.
* Rutas de visitas.
* Automatizaciones sin supervisión.
* Panel corporativo avanzado.
* Reconocimiento automático de productos.
* Modelos propios de machine learning.

Primero deberá comprobarse que el MVP resuelve el seguimiento comercial básico.

---

# Riesgos futuros

El crecimiento del sistema deberá controlar:

* Saturación de funciones.
* Datos incompletos.
* Recomendaciones poco confiables.
* Duplicación con Intelisis.
* Falta de adopción.
* Resistencia de los vendedores.
* Exposición de información comercial.
* Integraciones no autorizadas.
* Dependencia excesiva de la IA.
* Incremento innecesario de costos.

---

# Principio de evolución

El proyecto deberá crecer mediante ciclos:

Observar.

↓

Detectar un problema real.

↓

Diseñar una solución pequeña.

↓

Probarla.

↓

Medir resultados.

↓

Mejorar o descartar.

Nunca desarrollar una función únicamente porque técnicamente sea posible.

---

# Regla Suprema

La visión futura puede ser amplia, pero el desarrollo presente deberá permanecer enfocado.

Primero validar.

Después integrar.

Luego automatizar.

Finalmente escalar.
