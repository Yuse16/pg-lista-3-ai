# VENTAS.md

# Base de Conocimiento de Ventas

Proyecto: PG Lista 3 AI

Versión: 1.0

---

# Objetivo

Este documento define cómo debe interpretar la Inteligencia Artificial la información de ventas de Plomería García.

Las ventas representan el indicador más importante del negocio.

El agente utilizará esta información para detectar oportunidades, analizar el comportamiento de los clientes y ayudar al vendedor a incrementar la facturación.

---

# Filosofía

Una venta no representa únicamente una factura.

Representa una relación comercial.

Cada venta genera información valiosa para comprender:

* Qué compra el cliente.
* Cuándo compra.
* Cada cuánto compra.
* Qué marcas prefiere.
* Qué categorías consume.
* Qué oportunidades existen para vender más.

---

# Información de una Venta

Cada venta deberá contener como mínimo:

* Número de factura
* Fecha
* Cliente
* Vendedor
* Sucursal
* Productos
* Cantidades
* Precio unitario
* Descuento aplicado
* Subtotal
* IVA
* Total
* Forma de pago
* Lista comercial

---

# Origen de la Información

La información oficial de ventas provendrá de Intelisis.

La aplicación utilizará esta información únicamente para análisis, seguimiento e inteligencia comercial.

---

# Objetivos del Análisis

La IA deberá analizar cada venta para responder preguntas como:

* ¿Quién compró?
* ¿Qué compró?
* ¿Cuándo compró?
* ¿Qué dejó de comprar?
* ¿Qué podría comprar después?
* ¿Qué productos suelen venderse juntos?
* ¿Qué clientes están disminuyendo su frecuencia de compra?

---

# Indicadores Comerciales

El sistema calculará automáticamente:

* Ventas diarias
* Ventas semanales
* Ventas mensuales
* Ventas anuales
* Ticket promedio
* Número de ventas
* Número de clientes
* Productos vendidos
* Margen estimado (si está disponible)
* Crecimiento porcentual
* Comparativo contra periodos anteriores

---

# Ticket Promedio

El ticket promedio es uno de los indicadores más importantes.

La IA deberá buscar constantemente oportunidades para incrementarlo mediante:

* Ventas cruzadas.
* Productos complementarios.
* Productos premium.
* Paquetes de productos.

---

# Frecuencia de Compra

Para cada cliente el sistema calculará:

* Tiempo promedio entre compras.
* Última compra.
* Tendencia de compra.
* Variación en frecuencia.

Si la frecuencia disminuye, el agente deberá generar una alerta.

---

# Clientes Estratégicos

La IA identificará clientes estratégicos considerando:

* Volumen de compra.
* Frecuencia.
* Historial.
* Potencial de crecimiento.
* Participación en las ventas.

Estos clientes deberán recibir seguimiento prioritario.

---

# Clientes en Riesgo

Un cliente podrá considerarse en riesgo cuando:

* Deje de comprar durante más tiempo del habitual.
* Disminuya significativamente su volumen de compra.
* Existan cotizaciones sin seguimiento.
* Se detecte una reducción constante en su actividad.

El sistema deberá proponer acciones para recuperarlo.

---

El agente deberá conocer:

* Productos más vendidos.

* Marcas más vendidas.

* Productos con menor rotación.

# Predicción de Compras

Con suficiente información histórica, el sistema podrá estimar:

* Próxima compra esperada.
* Productos probables.
* Temporada de compra.
* Monto estimado.

Estas predicciones serán utilizadas únicamente como apoyo para el vendedor.

# Comparativos

La aplicación deberá permitir comparar:

* Día contra día.
* Semana contra semana.
* Mes contra mes.
* Año contra año.
* Sucursal contra sucursal.
* Vendedor contra vendedor.
* Cliente contra cliente.

---

# Objetivos Comerciales

El sistema deberá ayudar al vendedor a:

* Incrementar ventas.
* Incrementar ticket promedio.
* Incrementar frecuencia de compra.
* Recuperar clientes.
* Detectar nuevas oportunidades.
* Aumentar ventas complementarias.

---

# Inteligencia Artificial

La IA deberá responder preguntas como:

¿Qué clientes tienen mayor potencial?

¿Qué clientes dejaron de comprar?

¿Qué productos puedo recomendar?

¿Qué categorías están creciendo?

¿Qué clientes compran únicamente una línea de productos?

¿Qué ventas requieren seguimiento?

---

# Alertas Inteligentes

El sistema podrá generar alertas cuando:

* Un cliente importante deje de comprar.
* Una categoría disminuya sus ventas.
* Exista una caída significativa en el ticket promedio.
* Un producto aumente su demanda.
* Se detecten oportunidades de venta cruzada.

---

# Dashboard Comercial

El panel principal deberá mostrar:

* Venta del día.
* Venta del mes.
* Cumplimiento del presupuesto.
* Ticket promedio.
* Clientes nuevos.
* Clientes recuperados.
* Ventas por categoría.
* Ventas por vendedor.
* Ventas por sucursal.
* Oportunidades detectadas por la IA.

---

# Objetivo del Agente

El agente no deberá limitarse a mostrar cifras.

Su responsabilidad será interpretar los datos y convertirlos en acciones concretas.

Ejemplos:

"Cinco clientes importantes no han comprado este mes."

"Las ventas de porcelanato aumentaron un 18%. Es buen momento para impulsar adhesivos y boquillas."

"El ticket promedio disminuyó respecto al mes anterior. Se recomienda reforzar la venta complementaria."

---

# Integración con Intelisis

Toda la información de ventas deberá sincronizarse desde Intelisis.

La aplicación no modificará registros del ERP.

Su función será analizar la información y generar inteligencia comercial.

---

# Regla Suprema

Cada venta deberá responder tres preguntas:

* ¿Qué aprendimos del cliente?
* ¿Qué oportunidad comercial genera esta venta?
* ¿Qué acción puede realizar el vendedor para aumentar la siguiente compra?

Si una venta no genera conocimiento para mejorar la siguiente interacción, el sistema no estará aprovechando todo su potencial.

---

# Visión Final

Las ventas dejarán de ser únicamente un registro histórico.

PG Lista 3 AI transformará cada venta en conocimiento accionable, permitiendo que los vendedores anticipen necesidades, fortalezcan relaciones comerciales y aumenten las ventas mediante recomendaciones inteligentes basadas en datos reales.
