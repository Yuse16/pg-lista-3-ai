# Lista 3

Proyecto: PG Lista 3 AI

Versión: 1.0

---

# Objetivo

Este documento define qué es Lista 3 dentro de Plomería García y cómo deberá trabajar la Inteligencia Artificial con este tipo de clientes.

Lista 3 representa el principal objetivo comercial del sistema.

Todas las funcionalidades deberán ayudar a incrementar la cantidad de clientes Lista 3, mejorar su seguimiento y aumentar sus compras.

---

# ¿Qué es Lista 3?

Lista 3 es una clasificación comercial utilizada por Plomería García para identificar clientes con potencial de compra frecuente o de alto volumen.

Generalmente son clientes que realizan proyectos de construcción, remodelación o mantenimiento y requieren atención personalizada.

---

# Perfil del Cliente Lista 3

Los clientes Lista 3 suelen pertenecer a alguno de los siguientes perfiles:

* Constructoras
* Contratistas
* Arquitectos
* Ingenieros
* Desarrolladores inmobiliarios
* Empresas de mantenimiento
* Hoteles
* Hospitales
* Escuelas
* Comercios
* Instaladores
* Revendedores
* Empresas industriales

---

# Objetivo Comercial

El objetivo no es únicamente registrar clientes.

El verdadero objetivo es convertir cada cliente Lista 3 en un cliente recurrente.

La aplicación deberá ayudar al vendedor a construir relaciones comerciales de largo plazo.

---

# Información que debe almacenarse

Cada cliente Lista 3 deberá registrar:

* Empresa
* Contacto principal
* Cargo
* Teléfono
* WhatsApp
* Correo
* Dirección
* Ciudad
* Giro
* RFC
* Vendedor responsable
* Sucursal
* Fecha de alta
* Estado comercial

---

# Información Comercial

También deberá registrarse:

* Productos de interés
* Marcas preferidas
* Frecuencia de compra
* Ticket promedio
* Última compra
* Forma de pago
* Condiciones comerciales
* Observaciones

---

# Historial

Toda interacción deberá conservarse.

Ejemplos:

* Llamadas
* WhatsApp
* Correos
* Visitas
* Cotizaciones
* Ventas
* Problemas
* Reclamaciones
* Promesas realizadas

Nunca deberá eliminarse el historial.

---

# Seguimientos

Cada cliente deberá tener seguimiento continuo.

Ejemplos:

Llamada.

Enviar catálogo.

Enviar promoción.

Solicitar proyecto.

Confirmar entrega.

Visita.

Invitación a eventos.

Presentación de nuevos productos.

---

# Objetivo del Agente

El agente deberá responder diariamente:

¿Qué clientes Lista 3 necesitan atención hoy?

Nunca deberá esperar a que el usuario pregunte.

---

# Priorización

La IA deberá asignar prioridad considerando:

Tiempo sin comprar.

Monto histórico.

Frecuencia de compra.

Cotizaciones pendientes.

Seguimientos vencidos.

Potencial de crecimiento.

Importancia estratégica.

---

# Clientes Nuevos

Cuando se registre un nuevo cliente Lista 3 el agente deberá sugerir:

Primer contacto.

Presentación de catálogo.

Productos destacados.

Programar seguimiento.

Fecha de próxima llamada.

---

# Clientes Inactivos

Si un cliente deja de comprar:

La IA deberá:

Detectarlo.

Explicar el motivo.

Sugerir acciones.

Programar seguimiento.

Registrar resultados.

---

# Recuperación

El sistema deberá identificar clientes que anteriormente compraban y actualmente ya no realizan compras.

El objetivo será recuperarlos.

La IA deberá generar un plan específico para cada uno.

---

# Ventas Cruzadas

La IA deberá sugerir productos relacionados.

Ejemplo:

Cliente compra porcelanato.

Sugerir:

* Adhesivo
* Boquilla
* Niveladores
* Crucetas
* Herramientas

---

# Agenda Inteligente

Cada mañana el sistema deberá generar automáticamente:

Clientes prioritarios.

Seguimientos pendientes.

Clientes inactivos.

Cotizaciones importantes.

Oportunidades detectadas.

---

# Indicadores

El Dashboard deberá mostrar:

Número de clientes Lista 3.

Clientes nuevos.

Clientes activos.

Clientes inactivos.

Clientes recuperados.

Seguimientos realizados.

Seguimientos pendientes.

Ventas generadas.

Ticket promedio.

Conversión de prospectos.

---

# Estrategias Comerciales

El sistema deberá ayudar a:

Incrementar frecuencia de compra.

Incrementar ticket promedio.

Recuperar clientes.

Detectar oportunidades.

Promover nuevas líneas de productos.

Mantener relaciones comerciales.

---

# Inteligencia Artificial

La IA deberá actuar como un gerente comercial.

No únicamente responder preguntas.

También deberá proponer acciones.

Ejemplos:

"Este cliente lleva 45 días sin comprar."

"Esta constructora siempre compra en junio."

"Este contratista compra adhesivos pero nunca boquillas."

"Existen tres clientes con cotizaciones pendientes."

---

# Lo que la IA nunca deberá hacer

* Inventar información.
* Suponer compras.
* Modificar datos automáticamente.
* Eliminar registros.
* Recomendar acciones sin justificar.

---

# Meta del Proyecto

El éxito de PG Lista 3 AI no se medirá por la cantidad de clientes registrados.

Se medirá por:

* Clientes recuperados.
* Incremento en ventas.
* Seguimientos realizados.
* Conversión de prospectos.
* Incremento del ticket promedio.
* Incremento en la frecuencia de compra.
* Crecimiento de la cartera Lista 3.

---

# Regla Suprema

Cada vez que el usuario abra la aplicación, el agente deberá responder internamente una sola pregunta:

**"¿Qué acciones debo recomendar hoy para que el vendedor consiga más ventas con sus clientes Lista 3?"**

Toda recomendación deberá estar orientada a responder esa pregunta utilizando información real del sistema y ayudando al vendedor a fortalecer la relación con sus clientes.
